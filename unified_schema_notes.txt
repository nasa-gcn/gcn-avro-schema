From Erics email (13Sep21):
This looks like a great setup for instrument-specific streams. We should build an example for 'event streams' (i.e. GRBs or GWs), but here are some thoughts here
1) For the 'Reporter' fields we should figure out if we require a unique namespace or not. If we have to have unique entries for each field then we probably need 
   to work out the ordering, i.e. the first to report gets the _1 label, the second the _2 label. This may vary between streams (i.e. GBM could be _1 for the GRB 
   stream but _2 for the GW-GRB stream)
2) Leo requested we add an ellipse to the spatial options. We would need to pick a standard here. I think we also discussed adding a flag that would represent if 
   that representation was reasonably valid, which we could add as a fourth entry with a 1 for yes or 0 for no (which would imply they should use the skymap)
3) 'Distance' could be part of 'Spatial'
4) The GRB-specific entries could be all under one option, which would make it cleaner when considering multiple event types, but maybe this isn't good as different 
   information is reported at different stages (see next comment)
5) I wonder if for each subschema we should have an array that is the same length as the other entries with 1 indicating that entry is filled out and 0 if not? I 
   would think this could help follow-up code but we could see what Leo thinks.
